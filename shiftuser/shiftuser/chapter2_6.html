The SHIFT System Environment

System Disks

Each shift node has a system disk with traditional UNIX filesystems, / (root) and /usr, mounted from separate disk partitions. Each node has a backup system disk which can be used for booting in the event of a disk failure. The backup disk is cloned periodically from the primary system disk so as to propagate system changes.

/usr/local

By convention, the /usr/local filesystem contains all software and libraries that have been installed at CERN. This includes the CERN library, the SHIFT software, and a number of utility packages. There is a single copy of /usr/local which is shared across the shift nodes using NFS (Network File System) /usr/local has several sub-directories:

  • [src] source code
  • [lib] libraries
  • [bin] executable programs and scripts
  • [etc] Systems administration scripts and startup files

Home Directories

A user's home directory on SHIFT is defined by an entry in the /etc/passwd file and it is created when the user is registered. SHIFT home directories have the generic form: /u/gg/username, where gg is the 2-character group code and username is the user's login identifier.

The physical disk space used for the /u directory is made up of several separate filesystems, which are mounted as /u1, /u2 as shown in the Table below: 




--Filesystem----Machine-----Size-----Usage----------------------------------
  /u1           COREHOME    4.8GB    Home directories for CSF/SHIFT users

  /u2           SHIFT1      2.8GB    SHIFT home directories for OPAL
  /u3           SHIFT3      2.3GB    SHIFT home directories for ALEPH

In order to maintain a coherent naming convention, a series of symbolic links is maintained in the /u directory in all CORE systems. For example: 

/u/c3  --->  /u1/c3
     /u/ct  --->  /u1/ct
     /u/ws  --->  /u2/ws
     /u/xu  --->  /u3/xu
This shows that the home directory space for Group C3 is part of the /u1 filesystem residing on COREHOME whilst that of Group WS is part of /u2 located on SHIFT1. Home directories allocated on the COREHOME system are subject to disk quotas which control space allocation. To have your quota increased, contact one of the SHIFT systems adminitrators.

For client systems, the /u directory may be a local directory or may be mounted via NFS from one of the central servers such as COREHOME.

POOL Filesystems

In the shift terminology, a disk pool is a group of filesystems located on one or several disk servers. Each of these filesystems is mounted by NFS on every shift node using a standard naming convention, the nfs_pathname:

/shift/node/file_system_name/group/user/user_supplied_partial_pathname

where node is the machine on which the filesystem is physically located.
